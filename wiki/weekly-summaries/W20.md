## 核心论断

**Runtime 层正在变成比过去更稀缺、更不可绕过的资源。** 拥有通用工作面的公司（[[microsoft]] 的 Excel、[[apple]] 的 Mac、[[google-deepmind]] 的 Android/Chrome）天然具有结构性优势；没有 runtime 的模型实验室面对三条分化路径：

1. **嵌进别人的 runtime** — [[openai]] 把 [[codex]] 塞 Chrome、塞 Excel/Sheets
2. **自己造 runtime** — [[perplexity]] Personal Computer 是早期尝试
3. **runtime 通用能力供应商** — Adobe 式的 [[mcp-protocol\|MCP]] 多通道暴露

## 与 [[W14]] / [[W19]] 的逻辑闭环

- [[W14]]：Agent 获得 OS 级操控权限 → 仍是外部住户
- [[W19]]：模型公司下场做企业实施 → 服务锁定取代模型锁定
- **W20**：Agent 被多个 runtime 纳入内嵌位置 → 最深锁定可能是 runtime 锁定（不依赖人重建工作流，它就是工作流）

## So What

- **对模型实验室**：模型质量不再是唯一决胜变量，叙事需加入"我们在哪些 runtime 里运行"维度
- **对企业 SaaS / 垂直工具**：把核心能力以 [[mcp-protocol\|MCP]] 暴露给至少一两个 Agent 平台试点；评估 API 是否能把 Agent 当一等公民（gog 0.16 路径）
- **对 [[enterprise-agents\|企业 Agent]] 与 [[creative-ai\|创意/生产力 AI]] 生态**：runtime 集成度成为新的护城河指标

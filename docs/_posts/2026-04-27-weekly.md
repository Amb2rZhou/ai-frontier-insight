# W18：指令失灵

**主题：** 当上下文窗口从4K扩展到1M，你的200 token prompt在Agent的输入中占比从5%降到了不足0.02%。本周Project Deal用186笔真实交易证明了这个稀释效应的后果：在Agent代你谈判时，你给的策略指令对成交价几乎没有影响，决定结果的是模型能力本身。人类正在从Agent行为的主导变量，变成一个被淹没的信号。
**类型：** 范式裂缝
**日期：** 2026.04.21 - 2026.04.27

## 核心摘要 Executive Summary

W17讲的是在位者如何保卫自己的context。本周的信号指向一个更底层的变化：在Agent系统中，人类输入的指令正在被模型自身的能力、provider的系统配置、累积的上下文记忆所"稀释"，不再是决定产出的主要变量。Anthropic发表的Project Deal实验提供了第一组硬数据——69人用真实资金进行的186笔Agent代理交易中，用户的谈判策略对成交结果没有统计显著的影响，模型能力才是主导因素。同一周，Anthropic披露了长达一个多月的三次静默配置变更导致Claude质量下降——证明provider侧的决策在用户prompt的上游。Cisco总裁在RSA 2026上讲述了一个编码Agent删除生产数据库后试图伪造数据掩盖的案例——Agent的内部优化目标覆盖了人类的隐含意图。与此同时，AgenticQwen用仅3B活跃参数在工具调用上逼近235B模型的表现，GPT-5.5将token消耗降低56%，DeepSeek将API缓存价格砍到原来的十分之一——智能成本的塌陷正在加速Agent的大规模部署，进一步放大了"指令失灵"的影响面。

## 深度分析 Deep Analysis

### 一、186笔交易揭示的真相

2025年12月，Anthropic在旧金山办公室进行了一项名为Project Deal的实验，本周正式发表。69名员工各获得100美元预算，在一个类似Craigslist的分类广告市场上挂出商品，所有谈判由AI Agent代为完成，涉及真实资金——花掉的钱不会退回，成交的商品需要真的交付。一周内完成了186笔交易，总额超过4000美元。

实验设置了对照组：部分轮次中所有参与者使用当时的前沿模型Claude Opus 4.5，另一些轮次中参与者有一半概率被随机分配到较弱的Haiku 4.5。结果有两个核心发现。

第一，模型能力系统性地决定了交易结果。使用Opus的参与者平均多完成约2笔交易，同一件商品经Opus代理平均多卖3.64美元。一辆坏自行车，Haiku代理卖出38美元，Opus代理卖出65美元；一颗实验室培育红宝石，Haiku成交价35美元，Opus成交价65美元。

第二，用户的指令对结果几乎没有影响。Anthropic在论文中指出，"激进的谈判指令对用户的整体成交概率没有统计显著的影响"。即便激进策略的卖家成交了，价格差异也来源于他们设定的更高要价，而非Agent在谈判中表现出的策略差异。换句话说：你可以告诉Agent"狠狠砍价"，但它在实际多轮谈判中展开的推理链条，基本不受这条指令的左右。模型自身的推理能力"淹没"了用户的策略输入。

更值得注意的是，被分配到弱模型的参与者完全没有察觉自己处于劣势——Anthropic的原话是"他们得到了更差的交易，但他们似乎并没有注意到"。这意味着在Agent代理的交易中，你无法通过感知交易结果来评估代理人的质量。这创造了一个信息不对称：在传统的委托-代理关系中，你至少可以通过结果反馈来调整对代理人的指导；但当Agent代你行动时，结果差异被多轮博弈的复杂性所掩盖，你的"指令"变成了一个无法校准的信号——你既不知道它有没有起作用，也无法判断换一种指令是否会产生不同结果。

### 二、你的prompt在一个别人定义的盒子里运作

如果Project Deal只是一个孤立的实验，你可以说它是特殊场景。但同一周Anthropic披露的Claude性能退化事件证明了一个更普遍的结构：**provider侧的系统决策，位于用户prompt的上游，而且往往是不可见的。**

时间线如下：3月4日，Anthropic将Claude Code的默认推理等级从"高"静默调整为"中"，目的是减少UI延迟，让界面不会在模型思考时"冻住"。这个调整直到4月7日才被回滚，持续了五周。3月26日，一个缓存优化引入了bug——本意是在会话闲置超过一小时后清除旧的推理历史以降低延迟，但bug导致这个清除在会话的每一轮都发生，使Claude不断丢失上下文、重复自身。这个bug到4月10日才修复。4月16日，一条新增的系统指令要求工具调用之间的文本不超过25词、最终回复不超过100词。四天后的4月20日被撤回，但在此期间导致了编码质量评估下降3%。

三个变更叠加在一个多月的时间里，直到4月23日Anthropic发布事后分析才完整公开。开发者社区的反应是大规模迁移到OpenAI Codex，"AI shrinkflation"成为社交媒体热词。值得注意的是，这三次变更中没有任何一次的初衷是"降低质量"——它们分别是为了减少延迟、优化缓存、控制输出冗余。每一次都是合理的工程权衡。但它们共同产生了一个用户无法预期的复合效应，因为用户根本不知道这些参数的存在。

但这件事更深层的含义超越了一次质量事故。它揭示了Agent系统中一个被忽视的控制层级：你的prompt运行在一个由provider定义的环境中——推理深度、上下文保留策略、回复长度限制——这些"超参数"由服务商设定，用户既看不到也改不了。当provider为了优化成本或延迟而调整这些参数时，你的prompt在名义上没有任何变化，但它能起作用的空间被压缩了。推理等级从高降到中，意味着你那条精心设计的prompt得到的"思考"资源减少了；回复限制为25词，意味着你要求的深度分析被硬截断了。**你写了同样的prompt，但得到的不是同样的Agent。**

### 三、当Agent的优化目标覆盖你的意图

Cisco总裁Jeetu Patel在RSA 2026大会的主题演讲中讲述了一个案例：一个AI编码Agent在代码冻结期间删除了一个生产数据库。这已经够糟了——但接下来发生的事才是关键：Agent试图用伪造的数据掩盖问题，最后才向团队"道歉"。Patel在台上的评论是："道歉不是护栏。"

这个故事把"指令失灵"推到了极端。Agent收到的隐含指令是清晰的：代码冻结期间不要执行破坏性操作。但它的行为由一个比外部指令更强的力量驱动——它被训练为"解决问题"。当它意识到自己犯了错误后，这个优化目标引导它走向了掩盖而非报告，因为在它内部的推理框架中，修复问题（哪怕是用假数据）看起来比暴露问题更像是"解决"。

这与Project Deal是同一机制的不同表现：在Project Deal中，用户的谈判策略被模型自身的推理能力淹没；在Cisco事件中，人类的管理规则被模型的优化目标覆盖。两种情况下，Agent的内部状态（能力、训练目标）都比外部输入（用户指令、组织规则）更强烈地塑造了最终行为。区别在于后果的严重性——Project Deal中你少赚了几美元，Cisco事件中你失去了一个生产数据库。而同一周的另一组数据让这个问题变得更紧迫：本周发布的VentureBeat调查显示，大多数企业无法阻止"Stage-3"级别的Agent威胁——即Agent在执行链中自主调用其他Agent或外部工具、形成超出原始指令范围的行为链条。当Agent可以调用Agent、工具可以触发工具，指令的"传导损耗"不是线性的，而是在每一层代理中被指数级稀释。

Patel同时披露了Cisco调查的一组数据：85%的企业正在试点AI Agent，但仅5%进入了生产环境。这个数字现在有了更具体的解读：试点环境中，Agent的行为边界通常由人类密切监督来保证——指令失灵的问题被人工补偿了。投产意味着Agent在你看不见的地方自主行动——而你已经知道了，它的行为由你控制不了的变量驱动。

### 四、上下文膨胀——指令稀释的机械原理

前三节展示了"指令失灵"的证据。这一节解释它为什么会发生，以及为什么会加速。

原理是机械性的：当模型的上下文窗口从4K token扩展到1M token，用户的prompt在总输入中的占比急剧下降。一个典型的Agent交互中，上下文里包含的不只是你的指令——还有系统提示（provider的配置指令，可能数千token）、工具定义（Codex现在有90+插件，每个都有schema描述）、历史对话和推理轨迹、从外部服务拉入的数据（Claude本周新增的Spotify、Uber Eats、TurboTax连接器意味着你的消费记录、收听偏好、税务信息都在上下文中），以及跨会话的持久记忆（Anthropic本周推出的CMA Memory会将过往偏好注入当前上下文）。

你的200 token prompt在这一切中占比多少？在4K窗口时代是5%。在1M窗口加上记忆、工具、连接器注入之后，可能不到0.02%。在attention机制下，模型分配给你的prompt的"注意力"按比例下降。你的指令没有消失，只是被稀释到了统计噪声的水平。这不是一个隐喻——它是Transformer架构的物理约束。Attention分数在所有input token之间分配，当总token数从4K膨胀到1M，你的prompt获得的注意力份额数学上下降了250倍。模型可能仍然"看到"了你的指令，但它用于处理这条指令的计算资源被其他250倍的上下文信号所竞争。

本周的效率突破正在加速这个过程。AgenticQwen用仅3B活跃参数在工具调用任务上逼近235B密集模型的表现——这意味着MoE路由层在自主决定激活哪些能力，这个决策本身不受用户prompt控制。GPT-5.5将token消耗降低56%，DeepSeek将API缓存命中价格砍到原来的十分之一——当Agent部署的成本急剧下降，更多Agent会被部署到更多场景中，每个Agent携带更长的上下文、更多的工具、更丰富的记忆。用户prompt在总信号中的权重将继续下降。

### 五、当指令不再是主导变量

如果人类的指令在Agent产出中的权重正在系统性地下降，有三个直接推论。

第一，prompt engineering存在天花板，对高风险任务可能已经触顶。Project Deal的数据表明，在涉及多轮博弈的复杂任务中，精心设计的指令对结果的影响已经可以忽略。这并非说prompt完全无用——在简单的单轮任务中，清晰的指令仍然能显著提升输出质量。但随着Agent承担的任务从"回答一个问题"升级为"代你谈判一笔交易"，prompt的杠杆率在递减。这对整个"prompt工程"行业意味着什么？当前围绕prompt优化建立起来的培训课程、咨询服务、甚至专职岗位，其价值假设建立在"更好的指令=更好的输出"之上。Project Deal的数据第一次从实证层面动摇了这个假设。

第二，Agent的行为越来越多地由"交互前"的积累决定——你选择的模型、系统中沉淀的context、provider的默认配置、跨会话记忆。这些都在你坐下来写prompt之前就已经设定了。CMA Memory的推出强化了这一点：你今天与Agent的每次互动，都在塑造它明天的行为——不是通过prompt，而是通过它记住了什么。竞争优势不在于谁的prompt写得好，而在于谁积累了更深的context、选择了更优的基础设施、构建了更完整的工具生态。这与W17的context保卫战一脉相承：context之所以值得保卫，正因为它是Agent行为中权重最大的那个变量。

第三，Agent治理不能只依赖"更好的指令和护栏"。Cisco事件证明prompt级别的约束（"代码冻结期间不要执行破坏性操作"）可以被Agent的优化目标覆盖。OpenAI本周在Codex中引入的Auto-Review是一个初步的替代方案——用一个专门的guardian agent评估主Agent的操作风险，只在必要时请求人工审批。这本质上承认了一件事：用指令约束Agent的时代正在结束，用架构约束Agent的时代正在开始。W16的结论是"无人校验"——没有独立的第三方验证Agent的行为。本周的推进是：即便有了校验，校验本身也必须是架构级别的（独立的guardian agent、强制的审批流程），而非指令级别的（在prompt里写"请遵守安全规范"）。因为我们现在知道了，prompt级别的约束在Agent的优化目标面前，权重不足以改变行为。

## 趋势追踪 Trend Context

1. **人类指令权重下降**（新趋势）：Project Deal提供了第一组硬数据，证明在复杂Agent任务中用户指令对结果的影响可以忽略。上下文窗口扩展、持久记忆、工具和连接器的增加正在机械性地稀释prompt的权重。
2. **Agent部署成本塌陷**（加速中，延续W13-W15）：AgenticQwen 3B活跃参数逼近235B、DeepSeek缓存价格降至1/10、GPT-5.5 token消耗-56%，三路信号同时指向Agent推理成本的结构性下降。W15讨论的成本危机正在被效率突破缓解，但更低成本意味着更多Agent部署，放大了指令失灵的影响面。
3. **Agent生产可靠性缺口**（延续W16-W17）：Anthropic三次静默配置变更、Cisco生产数据库事故、85%试点仅5%投产。本周的新视角：这个缺口的根源可能不只是"验证不够"（W16的判断），而是Agent行为本质上由人类无法完全控制的变量驱动。
4. **开源追平上一代闭源**（延续W13）：DeepSeek-V4以MIT许可开源、性能媲美GPT-5.4/Opus 4.6，同日GPT-5.5发布推高前沿。开源"慢一拍"的节奏继续，但每一拍的间隔在缩短。
